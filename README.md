# Organic Acid Leaching Efficiency -- ML Pipeline

End-to-end machine learning pipeline for predicting the **leaching efficiency (%)** of metals (Co, Li, Ni, Mn) from spent lithium-ion batteries using organic acid systems.

---

## Setup

### 1 -- Prepare the dataset

The pipeline expects the raw CSV at `data/raw_dataset.csv`.
Copy or rename your augmented labelled dataset:

```bash
cp Full_Augmented_Dataset_Labeled.csv data/raw_dataset.csv
```

The file must contain at minimum:

- A `Source` column with value `"Real"` to identify real experimental rows
  (used as the held-out test set). All other rows are used for training.
  If this column is absent, a random 85/15 split is used as a fallback.
- An `Efficiency_pct` column as the regression target.

### 2 -- Configure environment variables (optional)

Copy `.env.example` to `.env` and edit as needed:

```bash
cp .env.example .env
```

Available variables:

| Variable     | Default     | Description                                      |
|--------------|-------------|--------------------------------------------------|
| `HOST`       | `127.0.0.1` | Flask bind address (`0.0.0.0` for Docker/cloud)  |
| `PORT`       | `5000`      | Flask port                                       |
| `FLASK_DEBUG`| `0`         | Set to `1` for development debug mode            |

## Usage

### Run the full training pipeline

```bash
python -m src.pipeline.training_pipeline
```

Or use the console script installed by `setup.py`:

```bash
train
```

This runs all four stages in sequence and writes artifacts to `artifacts/`.

### Launch the Flask web app

```bash
python application.py
```

Open `http://localhost:5000` in your browser. Fill in the leaching conditions
and click **Predict Leaching Efficiency** to get a result.

The form exposes all required input fields, including molecular descriptors,
functional group flags, and EHS sustainability scores. The EHS Total field is
auto-computed from the three component scores but can be overridden manually.

### REST API

The `/predict` endpoint also accepts JSON:

## ML Models Compared

| Model             | Notes                                          |
|-------------------|------------------------------------------------|
| Random Forest     | Robust ensemble; handles nonlinearity          |
| XGBoost           | State-of-the-art tabular regression            |
| LightGBM          | Faster XGBoost; good for smaller datasets      |
| Ridge Regression  | Linear baseline with L2 regularisation         |
| SVR (RBF kernel)  | Strong generaliser on small real test sets     |

**Selection criterion**: highest Test R2 on the **real-data test set only**.
Synthetic rows are never used for evaluation. The minimum acceptable Test R2
is 0.60; training raises an error if no model clears this threshold.

## Features Used

**Experimental:** Concentration (M), Temperature (C), Time (hrs), Solid-Liquid Ratio (g/L), Has Reductant

**System:** Solvent Type, Battery Chemistry, Reductant Type, Target Metal

**Molecular (RDKit):** MW, LogP, TPSA, H-Bond Donors/Acceptors, Rotatable Bonds, Heavy Atoms, Morgan FP Density, functional group flags (Carboxyl, Hydroxyl, Halogen, Phosphorus, Ionic)

**Sustainability (EHS):** Environment, Health, Safety, Total, GreenScore

**Engineered** (computed in `src/utils/features.py`, shared between training and inference):

| Feature         | Physical motivation                                     |
|-----------------|---------------------------------------------------------|
| `log_Time_hrs`  | Leaching kinetics follow log-time diminishing returns   |
| `log_SLR_gL`    | Solid-liquid ratio effect is sub-linear                 |
| `log_Conc`      | Acid dissociation contribution is ~log(concentration)   |
| `Conc_x_Temp`   | Synergistic interaction between acid strength and heat  |
| `Temp_x_logTime`| Arrhenius temperature x log-time interaction term       |
| `inv_Temp_K`    | 1000/(T+273.15) -- direct Arrhenius activation term     |

## Design Notes

- **No data leakage**: the preprocessor (StandardScaler, OneHotEncoder) is
  fitted on training rows only and applied to the test set without refitting.
- **Single source of truth for features**: all feature engineering lives in
  `src/utils/features.py` and is imported by both the training pipeline and
  the inference pipeline, preventing train/serve skew.
- **Input validation**: `LeachingInput.__post_init__` checks physical bounds
  (positive concentration, non-negative temperature, binary flags in {0,1}, etc.)
  before any model call is made.
- **Artifact safety**: `PredictPipeline` checks that model files exist and
  raises a clear `FileNotFoundError` with setup instructions if they are missing.
